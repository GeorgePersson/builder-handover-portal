# UX Rules

## Builder Review Before Publishing

Builders must be able to review and correct extracted items before anything is client-facing. Approval, exclusion, and builder-supplied states should be clear.

## Clean Client Views

Client-facing views should be simple, calm, and homeowner-safe. Show useful handover information, not internal AI reasoning, raw extraction text, or review queue noise.

## Clear Categories

Items should be grouped by practical homeowner/builder categories. AI-suggested categories are helpful, but builders need the final approved category control.

## Source And Reference Visibility

Source page, section, snippet, quote reference, supplier document, or warranty source should be visible where it helps review. Do not clutter homeowner views with unresolved source mechanics.

## Safe Editing

Editing should feel reversible and auditable. Preserve original extracted values beside current builder-edited values where useful. Capture review reasons for low-confidence or meaningful changes.

## Avoid Clutter

Prefer restrained dashboard/workspace screens over marketing-style pages. Avoid burying the builder in every extracted field at once; use lanes, status labels, and focused edit forms.

## Simple Controls

Use simple forms, clear labels, readable status chips, and direct actions. Avoid ambiguous buttons such as "fix with AI" unless the user can see what will change and review the result.

## AI Uncertainty

Show confidence, missing fields, source quality, quote-needed state, and review reasons in builder/admin flows. Never imply AI has fully solved an item when source quality is poor or identity is incomplete.

## Supplier And Manufacturer

Do not present supplier as manufacturer or vice versa. Builders should be able to change supplier without rewriting the product identity.

## Spec-Upload-First Workflow

The main builder path should start from project/spec/quote/manual upload and review. Manual product entry can exist as a fallback or admin tool, but it should not become the primary UX.
